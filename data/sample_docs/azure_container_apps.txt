Azure Container Apps is a serverless platform for running containerized applications and microservices. It is built on Kubernetes together with the open-source KEDA, Dapr, and Envoy projects, but it hides that complexity from the developer. Applications scale automatically based on HTTP traffic, event sources, or CPU and memory load. When there is no traffic, an app can scale to zero replicas, so you pay nothing for idle capacity. KEDA, the Kubernetes Event-Driven Autoscaler, provides the scaling rules that let an app react to queue length, HTTP concurrency, and custom metrics. Container Apps supports HTTPS ingress, revisions for blue-green and canary releases, secrets, and managed identities for talking to other Azure services. It is a good fit for REST APIs, background workers, and model-serving endpoints that need to scale out under load and scale back to zero when idle.
